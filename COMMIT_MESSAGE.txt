Fix PostgreSQL compatibility issues (MySQL to PostgreSQL migration)

This commit addresses all PostgreSQL compatibility issues found during
the MySQL to PostgreSQL migration. Three critical categories of issues
were identified and resolved:

1. CHECK Constraint Violation - Media Type
   - Fixed empty string ("") being inserted into media_type column
   - PostgreSQL CHECK constraint only allows 'image', 'video', or NULL
   - Solution: Default to 'image' when media_type is empty
   - File: internal/newincident/service.go

2. CHAR(1) Boolean Comparisons with Trailing Spaces
   - PostgreSQL CHAR(1) columns may contain trailing spaces (e.g., "1 ")
   - Direct comparisons like `is_active = '1'` fail with "1 "
   - Solution: Wrap all boolean comparisons with TRIM() function
   - Files: 8 repository files (15 instances total)
   - Pattern: `WHERE is_active = '1'` → `WHERE TRIM(is_active) = '1'`

3. MySQL DATE() Function Incompatibility
   - MySQL DATE() function doesn't exist in PostgreSQL
   - Solution: Replace with PostgreSQL ::date cast syntax
   - File: internal/referrals/repository.go (4 instances)
   - Pattern: `DATE(registered_at)` → `registered_at::date`
